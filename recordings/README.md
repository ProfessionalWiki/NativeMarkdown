# Native Markdown — launch demo (draft for review)

Draft deliverable: a demo video for the Native Markdown launch, plus a still, a GIF, and a reproducible
recipe. **Nothing is published.** Review the assets, then decide what ships where.

`native-markdown-demo.md` is the full re-record recipe. This file is the 30-second orientation.

## What's here

- `assets/native-markdown-demo.webm` / `.mp4` / `-poster.webp` — the hero video (WebM primary + MP4
  fallback + poster). **1280×800, ~29 s.** This is the blog embed.
- `assets/native-markdown-demo-hero.png` / `.webp` — a composed split still (rendered page ½ · Markdown
  source ½). The mediawiki.org screenshot + blog og:image.
- `assets/native-markdown-demo.gif` — ~12 s captioned teaser (~2.5 MB) for the mediawiki.org page.
- `assets/*.png` — supporting stills (rendered page, editor source, the Monitoring section, the blue link,
  the category bar, `action=raw`, the REST `content_model: markdown` response).
- `assets/native-markdown-demo.nocaps.mp4` — uncaptioned master (re-burn captions from this).
- `capture/*.mjs`, `captions.ass`, `stage.sh`, `make-video.sh` — the pipeline. Everything above is
  regenerated by the commands in `native-markdown-demo.md`; no screen recorder, no human at the keyboard.

## The video (what it says)

Reveal-forward, thesis in the first 2 s, ~29 s, silent with burned-in captions:
**a normal MediaWiki page → "there's no wikitext behind it" → Edit shows clean Markdown → type a section
(heading, link, category) → Save → the ToC grows, the link works, the category lands → `action=raw` is the
same Markdown, byte for byte → end card.** The AI angle is one closing line, not the subject.

## Choices, and why

| Decision | Choice | Why |
|---|---|---|
| Approach | One deterministic **browser screencast** (dropped the MCP/agent-terminal idea) | Keeps it about the extension, not a second product; and I can produce the whole thing — no human capture step. |
| Dimensions | **1280×800** | Matches the house UI-video size (NeoWiki clip) and the extension's own screenshots; renders at ~925 px in the blog column with HiDPI headroom. |
| Format | WebM (VP9, crf 32) + MP4 (H.264, crf 21) + WebP poster | The professional.wiki house trio; text stays crisp, each lands ~1.7–2.2 MB. |
| Length | **~29 s** (25 s flow + 4 s end card) | Long enough to show the round-trip, front-loaded so the point lands in the first few seconds. |
| mediawiki.org | still + GIF + link (not self-hosted video) | Verified: TimedMediaHandler blocks MP4, and wikitext can't embed a raw `<video>`; a still + GIF is the low-friction norm. WebM-on-Commons is the optional richer path. |
